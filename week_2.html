01
What Is SSH?
Secure Shell — remote control over an encrypted connection

SSH stands for Secure Shell. It lets you control a computer that is somewhere else — across the room, across the building, or across the world — using only your keyboard and a terminal window.

Encryption

Everything you type and everything the remote computer sends back is scrambled so that nobody in between can read it. Imagine passing a note in class written in a secret code that only you and your friend know — that is basically what SSH does for computers.

Why should you care?

If you want to work in cybersecurity, you will use SSH all the time. It is how professionals log into servers, investigate security problems, and manage systems. Learning SSH now is like learning to use a stethoscope if you want to be a doctor — one of the first tools you pick up.

02
How Does It Work?
The four steps from your terminal to the remote machine
  1. You tell your computer: "I want to connect to that computer over there."
  2. The two computers meet and agree on a secret code to use for the session.
  3. You prove who you are — usually by typing a password.
  4. Once you are in, you have a command line on the remote computer. You can type commands just like you would on your own machine.
Always encrypted

Everything that travels between the two computers is encrypted the entire time — login, commands, responses, everything.

03
The SSH Command
Breaking down every piece of what you type
ssh bandit0@bandit.labs.overthewire.org -p 2220
ssh
The program you are running — the SSH client.
bandit0
The username you are logging in as on the remote machine.
@
Separator — just separates the username from the address.
bandit.labs…
The address (hostname) of the computer you are connecting to.
-p 2220
The port to connect on. The default SSH port is 22 — this server uses 2220 instead.
Think of a port like an apartment number.

The address gets you to the building; the port number gets you to the right door. SSH defaults to door 22 — some servers use a different one for security or configuration reasons.

04
What Is OverTheWire Bandit?
A free, beginner-friendly wargame for learning Linux and SSH

Bandit is a free online game designed to teach you how to use the Linux command line. It is hosted at overthewire.org.

  • You log in to the server using SSH
  • Each level gives you a challenge — solve it to find a password
  • Use that password to log into the next level
  • The challenges get harder as you go
BEFORE YOU START — Make a Notes File

Bandit does NOT save your progress. If you find a password and forget to write it down, you have to start over. Right now, open a text editor and create a file called bandit_notes.txt. Paste every password into this file as you find them.

L0
Lab: Level 0 — Log In for the First Time
Goal: Connect to the Bandit server using SSH
Connection Info
Bandit Level 0
Host: bandit.labs.overthewire.org Port: 2220 Username: bandit0 Password: bandit0
  1. Open a terminal on your machine.
  2. Type this command and press Enter:

    ssh bandit0@bandit.labs.overthewire.org -p 2220
  3. The first time you connect, it will ask if you trust this server. Type yes and press Enter.
  4. It will ask for a password. Type bandit0 and press Enter.
  5. IMPORTANT: When you type the password, nothing will appear on the screen — no dots, no stars, nothing.
    This is normal.

    Linux hides your password so nobody looking over your shoulder can see it. Just type it and press Enter.

  6. If it worked, you will see a welcome message and then a prompt like this:
    bandit0@bandit:~$
    That means you are now on the Bandit server. You just completed Level 0.
L1
Lab: Level 0 → Level 1 — Find the Password
Goal: Read the readme file to get the password for the next level
Commands you need

ls — lists the files in your current folder  ·  cat filename — shows what is inside a file

  1. Make sure you are still logged in as bandit0 (your prompt says bandit0@bandit).
  2. Type ls and press Enter. You should see a file called readme.
  3. Type this and press Enter:

    cat readme
  4. A long string of random characters will appear. That is the password for bandit1. Copy it and paste it into your bandit_notes.txt file right now.
  5. Type exit and press Enter to disconnect from the server.
  6. Now log in as bandit1 using the password you just found:

    ssh bandit1@bandit.labs.overthewire.org -p 2220
  7. When it asks for a password, paste or type the password you saved.
  8. If your prompt now says bandit1@bandit:~$ then you did it.
    Level 0 → 1 complete!

    You used SSH to log in, found a file with ls, read it with cat, saved the password, and logged in as the next user. That is real-world sysadmin and security work.
