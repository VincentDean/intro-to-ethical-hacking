OVR
What You're Building
A working website that documents everything you've learned

You are going to design and build a multi-page website that works as a personal reference guide for ethical hacking. Think of it as your own knowledge base — the kind of technical documentation that real cybersecurity teams actually maintain.

Your site needs to be functional, readable, and accurate. Every section has specific requirements, but how you design it — the layout, colors, and structure — is entirely up to you. The finished product should be something you'd include in a portfolio.

Work in your own words

Do not copy definitions from Google or any other website. You have done the labs. You have run the commands. Your own understanding — written clearly — is the entire point of this assignment.

Multi-Page
One page per tool + homepage
Build separate HTML files for each tool. Link them together with a consistent navigation menu on every page.
Styled with CSS
Not just default browser styles
Apply a consistent visual theme across all pages. The design is your choice — just make it intentional.
Portfolio Ready
Something you'd show an employer
Write clearly for a reader who is smart but not a cybersecurity expert. Explain, don't just name-drop tools.
Bonus: JavaScript
Interactivity = extra credit
Any JavaScript feature that improves the experience earns bonus points on top of your base score.

03
Nmap — Network Mapper
Discovering what's on a network and what's running on each host
Required
What this page must include
  • A definition of Nmap and what "network scanning" means in plain terms
  • An explanation of what ports are and why knowing which ones are open matters to a security tester
  • A real Nmap command you have run, plus a sample of the output with the results explained
Example
The kind of thing your page might show
# Scan a subnet and detect service versions nmap -sV 192.168.1.0/24 PORT STATE SERVICE VERSION 22/tcp open ssh OpenSSH 8.4 80/tcp open http Apache 2.4.51 3306/tcp open mysql MySQL 5.7.36 # Port 3306 open = database exposed on the network. # That's a finding worth investigating.

Use your own commands and your own output. Include an explanation of what the results actually mean.

Wireshark — Packet Analyzer
Capturing and reading network traffic in real time
Required
What this page must include
  • An explanation of what a packet is — use an analogy if it helps you explain it clearly
  • A description of what Wireshark lets you see and why that is useful during a security investigation
  • A specific example: what kind of data or traffic could a tester find using Wireshark?
Tip
Think about unencrypted traffic
  • What happens when a user submits a login form over plain HTTP instead of HTTPS? What would Wireshark reveal?
  • Wireshark doesn't use terminal commands — you can describe what a packet capture looks like instead of showing a command.

Hydra — Credential Brute-Force Tool
Automated testing of username and password combinations
Required
What this page must include
  • An explanation of what Hydra does and why authorized security testers use it
  • A description of what a wordlist is and how Hydra uses it to test credentials
  • The Hydra SSH command from your lab, with each flag explained in plain English
Ethics — Required
This section is not optional
  • Explain clearly why running Hydra against a system without permission is illegal — not just against the rules
  • Describe what an organization can do to defend against brute-force attacks (account lockouts, rate limiting, MFA)
Example
The kind of thing your page might show
# Authorized credential test against an SSH service hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.50 # Flag breakdown: -l admin the username being tested -P rockyou the wordlist of passwords to try ssh://... the protocol and target IP [22][ssh] host: 192.168.1.50 login: admin password: password1 # Weak password found. Document it. Report it.
06
Brute-Force Attacks
The methodology behind credential-based attacks
This is a concept page, not a tool page

Brute-force is an attack method, not a single tool. Your job is to explain the idea itself — then show how tools like Hydra carry it out.

Required
What this page must include
  • A definition of a brute-force attack as a method — trying every possible combination until one works
  • The difference between a brute-force attack (all combinations) and a dictionary attack (a known wordlist)
  • An example scenario: where and how this type of attack is used during a penetration test
Ethics — Required
Defenses against brute-force attacks
  • Explain at least three defenses organizations use: account lockouts, rate limiting, multi-factor authentication
  • Describe what a penetration tester's finding would look like: what's the risk, and what's the recommendation?

Bonus Points — Interactivity
JavaScript features that improve the user experience
These are not required — but they push your grade higher

Any JavaScript feature that genuinely improves how a user experiences your site can earn bonus points. You may combine multiple ideas. Maximum bonus: +25 points on top of your base score.

Command Builder / Explainer +5–10 pts

A form where the user selects flags and sees a generated Nmap or Hydra command explained in real time using JavaScript.

Fake Terminal Window +5–10 pts

Quiz or Flashcards +5 pts

A multiple-choice or flip-card quiz that tests definitions and tool usage — built with JavaScript state logic.

Dark / Light Mode Toggle +5 pts

A button that switches the entire site between a dark and light color scheme using a JavaScript class toggle and CSS variables.

A search bar that filters content or highlights matching terms across your tool sections as the user types.
