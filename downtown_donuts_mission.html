Phase 1

Get on the Network

Phase 1 Primer — Key Concepts
TermWhat It Means
SSID The name of a Wi-Fi network. When you see a list of available networks, each name is an SSID.
IP address A unique number that identifies your device on a network — like a mailing address for your computer. Example: 192.168.1.45
Subnet The range of IP addresses that belong to the same local network. If your IP is 192.168.1.45, your subnet is probably 192.168.1.0/24.
ip a A Linux command that shows your network interfaces and their IP addresses.
ip route Shows you the network routes on your machine — useful for confirming your gateway and subnet.
Your Tasks
  1. You have a password sheet with several candidates. Only one works right now.
  2. Try each password until you connect to the Auralab SSID.
  3. Once connected, run the commands below to find your IP address.
  4. Record your IP address and subnet in the Mission Log.
ip a ip route
Checkpoint
Mission Log shows: successful password, confirmed SSID, your IP address, and your subnet.
Phase 2

Hunt the Network

Phase 2 Primer — Key Concepts
TermWhat It Means
nmap Network Mapper. A free tool used to scan networks and discover what devices and services are running. One of the most important tools in a security consultant's kit.
host Any device connected to a network — a computer, phone, printer, server, or Raspberry Pi.
CIDR notation /24 means the first 24 bits of the address are fixed — so 192.168.1.0/24 covers every address from 192.168.1.1 through 192.168.1.254.
-sn flag Tells nmap to do a ping scan only — no port scanning. Fast way to find which hosts are alive on the network.
-sV flag Tells nmap to detect what software is running on each open port. Slower, but gives you much more detail.
open port A numbered channel on a device. An open port means a service is actively listening there and will respond to connections.
Nmap Quick-Start Guide
Your first time using nmap — read this before opening a terminal
Step 1 — Open a terminal on your Kali machine

Right-click the desktop and select Open Terminal, or find it in the taskbar.

Step 2 — Build your scan range

When nmap scans a network, it needs to know which IP addresses to check. A network can have up to 254 devices, each with their own IP address.

Take the IP address you recorded in Phase 1. Keep the first three numbers exactly as they are, then replace the last number with 0/24. This tells nmap to check every possible address on your network.

Your IP:         10.0.0.47
Your scan range: 10.0.0.0/24

This range covers 10.0.0.1 all the way through 10.0.0.254 — every device that could be on your network.

Step 3 — Run a host discovery scan

This tells you which devices are alive on the network. It does not probe anything deeply.

nmap -sn 10.0.0.0/24 # replace with your actual subnet
Step 4 — Look for interesting hosts

nmap will list every device it finds. Your own machine and the router will appear — set those aside. Anything else is worth investigating.

Step 5 — Dig deeper on a candidate

Run a service scan on any host that looks unfamiliar. This reveals what is running on each open port.

nmap -sV 10.0.0.23 # replace with your candidate's IP
Your Tasks
  1. Use your IP from Phase 1 to build your scan range.
  2. Run a host discovery scan to find all active devices on the network.
  3. List everything you find. Look for hosts that seem unusual or out of place.
  4. Pick your top candidates and run a service scan to see what they are running.
Checkpoint
Mission Log lists all active hosts found, your top candidates, and your reasoning for choosing them.
Phase 3

Find the Leak

Phase 3 Primer — Key Concepts
TermWhat It Means
web service A program running on a device that serves web pages or data over HTTP. Dashboards, admin panels, and data feeds are all web services.
port 5000 A common port used by development web servers and data dashboards. Not a default — you have to specify it in your browser.
URL format To reach a specific port on a target device: http://<target-ip>:5000 — replace <target-ip> with the IP address you found in Phase 2.
Your Tasks
  1. Review your Phase 2 scan results and identify the target device.
  2. Note that device's IP address — not your own.
  3. Type the target's address into your browser using port 5000.
  4. Document what the dashboard is showing and capture your evidence.
http://<target-device-ip>:5000 # type this into your browser address bar
Checkpoint
Mission Log has the target device IP, port number, dashboard URL, and a screenshot of what is exposed.
