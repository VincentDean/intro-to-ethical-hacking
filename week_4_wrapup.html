Week in Review

This week covered four connected missions inside the Downtown Donuts scenario, taking teams from bare-metal system setup through live network surveillance, active host discovery, and credential-based access to a sensitive target.

Activity 1 — Kali Linux Boot & Exploration
CategorySummary
ObjectiveBoot Kali Linux from USB in Live Mode and explore the OS environment before any active testing begins.
Key TasksBoot via F12 menu, troubleshoot Secure Boot errors, locate system tools, identify 5 security tools in the Applications menu, run terminal commands, explore 2 tools of choice.
Tools FoundNmap  Wireshark  Metasploit  Burp Suite  John the Ripper
Terminalwhoami  pwd  ifconfig
Lab ContextPre-engagement environment verification for Downtown Donuts security consulting work.
Key TakeawayReal security analysts configure and validate their tools before any active testing begins. Technical problems during setup are part of the job — and documenting them is too.
Activity 2 — Network Recon: Find the Leak
CategorySummary
ObjectiveConnect to the Auralab Wi-Fi, scan for all active devices, and locate a data dashboard leaking sensitive information.
Key TasksConnect using a password list, confirm your IP with ip a and ip route, run a ping scan, identify suspect devices, run a service scan, access the dashboard on port 5000.
Commandsnmap -sn <subnet>  nmap -sV <ip>
New ConceptsSSID, IP address, subnet, CIDR notation, open ports, web services, port 5000.
DeliverableMission Log with all hosts found, target device IP, dashboard screenshot, and a complete Finding Report: Finding / Evidence / Impact / Recommendation.
Key TakeawayAttackers use the same recon tools defenders do. Reading nmap output accurately is a core analyst skill.
Activity 3 — Wireshark Traffic Analysis
CategorySummary
ObjectiveCapture live network traffic on Auralab and identify what two active devices are doing.
Key TasksConnect to Auralab, open Wireshark, select the wlan0 interface, let it run, identify two active IPs from the Source/Destination columns, interpret the Protocol column.
ProtocolsARP who's on the network  HTTP web data  TCP two-way conversation  UDP one-way stream  ICMP ping/alive checks
DeliverableCompleted observation sheet — interface selected, two device IPs identified, dominant protocol recorded, behavioral inferences written, reflection complete.
Key TakeawayPassive monitoring reveals real-time activity without touching any target. Defenders watch traffic to catch anomalies before they become incidents.
Activity 4 — Brute Force Grade Assignment
CategorySummary
ObjectiveSSH into a target device using a common-password brute force, navigate the filesystem, and open the grades document using Nano.
Key TasksRun ifconfig for your IP, run nmap -sn to find the grade server, try the top 10 common passwords over SSH, navigate to the grades file, open with nano.
Commandsifconfig  nmap -sn  ssh user@ip  nano <filename>
CriticalWrong moves in Nano can corrupt the file. Understand Ctrl+X / Y / Enter before editing anything.
Key TakeawayWeak passwords are the most common attack vector in real breaches. This exercise shows exactly why password policy matters.

Key Concepts — Quick Reference
TermPlain English
SSIDThe name of a Wi-Fi network as it appears in the connection list.
IP AddressA unique number identifying your device on a network — like a mailing address for your computer.
Subnet / CIDRThe range of IPs on your network. /24 covers .1 through .254 — up to 254 devices.
nmapNetwork Mapper. Discovers live hosts and identifies services running on open ports.
nmap -snPing scan only — fast way to see which hosts are alive, no deep probing.
nmap -sVService scan — reveals what software is running on each open port.
Open PortA numbered channel where a service is actively listening for connections.
WiresharkCaptures and displays every packet moving across a selected network interface.
ARPAddress Resolution Protocol — devices asking who else is on the network.
HTTPHypertext Transfer Protocol — web page data, sometimes readable as plain text.
TCPTwo devices having a structured back-and-forth conversation.
UDPOne device sending data without waiting for a reply.
ICMPPing messages — one device checking if another is alive.
SSHSecure Shell — encrypted remote login to another machine via terminal.
Brute ForceTrying every password candidate systematically until the correct one is found.
nanoA simple terminal text editor. Ctrl+X to begin exit, Y to confirm save, Enter to write.
Live ModeBooting an OS from USB without installing it. Nothing persists to the host machine.
