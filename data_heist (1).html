The Data Heist

Someone has been skimming money through the Downtown Donuts POS system. A rogue transaction was planted inside the live data stream. Your team's mission: find it, identify it, and report the transaction ID.

Target: Raspberry Pi 4
Network: Auralab
Tools: nmap · Wireshark · SSH
SSH User: ddadmin
// Mission Brief

A Raspberry Pi on the Auralab network is broadcasting Downtown Donuts point-of-sale transactions over HTTP. Most are normal — coffee, donuts, small amounts.

One transaction does not belong. The amount is way too high for a donut shop. That transaction ID is your flag.

Your three moves: Watch the traffic to spot the anomaly. Connect to the Pi via SSH. Find the transaction in the log file and submit the ID.

1
Find the Pi

The Pi is somewhere on the Auralab network. Use nmap to find it.

Open the Terminal and scan the network:

$ nmap -sn 192.168.1.0/24

Look through the results for a Raspberry Pi. It will appear as a live host. Write down its IP address — you will need it for every step that follows.

HINT Raspberry Pi devices often show "Raspberry Pi Foundation" in the nmap vendor field.

2
Watch the Traffic

Open Wireshark and start capturing on wlan0 or wlan0mon. The Pi is broadcasting transaction data every few seconds.

Filter for traffic from the Pi only. Replace PI_IP with the address you found:

// Wireshark filter bar — type this and press Enter
ip.addr == PI_IP

Watch the Info column. You will see HTTP requests going out. Most transactions will be small dollar amounts. Watch for the one that is not.

HINT Right-click any HTTP packet and choose Follow → HTTP Stream to read the full transaction data as plain text.
NOTE The unusual transaction will appear once during the broadcast. Keep watching — do not close Wireshark early.

3
SSH Into the Pi

Now that you know the Pi's IP, connect to it using SSH. Use the credentials from your Password List handout.

$ ssh ddadmin@PI_IP

Type the password when prompted. You will not see characters appear as you type — that is normal. Press Enter when done.

When you are in, your terminal prompt will change to show the Pi's hostname.

HINT Connection refused? Double-check the IP address. Permission denied? Check the password on your handout carefully — it is case sensitive.
ItemTeam Answer
What does the prompt show once you are logged in?
4
Find the Log File

The Pi is writing every transaction to a log file. Navigate to the logs folder and look inside.

$ cd ~/logs
$ ls
$ cat transactions.log

The log will show every transaction that has been broadcast. Scroll through and find the one with the suspicious amount you spotted in Wireshark.

HINT Too many lines to scroll through? Use grep to search directly:
grep "9847" ~/logs/transactions.log

5
Capture the Flag

You found the transaction. The txn_id field on that entry is your flag. It looks like this:

// Example format — yours will be different
"txn_id": "DD-XXXXXXXX-XXXX"

Write it down exactly, then bring it to your teacher to submit.
