The Correct Hydra Command

Copy this exactly. The IP is already set for this lab. Do not modify anything.

// Run This

  • hydra -l jrogers -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.115 -t 1 -W 3 -V Full command — all flags required for this hardware.
Flag What it does
-l jrogers Single username to attack (lowercase L)
-P rockyou.txt Wordlist of passwords — UPPERCASE P required
ssh://192.168.1.115 Target protocol and IP address
-t 1 1 thread — prevents the Pi from dropping the connection
-W 3 3 second wait between retries — keeps the session stable
-V Verbose — shows each attempt so you know it's running
Troubleshooting — Work Through In Order
01
Hydra printed a help menu / list of options
The command had a syntax error. Check: lowercase -p vs uppercase -P — they do different things and the wrong one breaks the command silently. Also confirm you replaced TARGET-IP with the actual address from your Nmap scan, and verify the wordlist path with ls /usr/share/wordlists/rockyou.txt.
02
rockyou.txt not found
Live USB does not persist files. Run sudo gunzip /usr/share/wordlists/rockyou.txt.gz at the start of every session. If it says the file already exists, skip it. Confirm with ls -lh /usr/share/wordlists/rockyou.txt — should show ~130MB.
03
[ERROR] 1 target was disabled — too many errors
The Pi is dropping connections because Hydra is hitting it too fast. The -t 1 -W 3 flags in the command above fix this. Make sure both flags are present. Do not run the default Hydra command without them on this hardware.
04
Hydra is running but counting slowly
Normal. letmein is near the top of rockyou.txt but the exact position varies slightly by Kali version. If the counter passes 100 with no result, raise your hand.
05
SSH says Permission denied
Write down the password Hydra prints exactly as shown. When entering it in SSH, nothing appears on screen while you type — this is standard Linux behavior. Type the password and press Enter.
