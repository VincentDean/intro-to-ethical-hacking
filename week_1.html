01
What Is the Command Line Interface?
Your First Steps in the Terminal

The Command Line Interface (CLI) is a text-based way to interact with a computer. Instead of clicking buttons and menus (GUI), you type commands and the system responds with text.

Who uses the CLI?

Cybersecurity professionals, software developers, and system administrators use the CLI every day. Learning it early gives you direct control over how systems work.

Speed
Many tasks are faster to type than to click.
Power
Some actions only exist in the CLI — no buttons, no menus.
Automation
Commands can be scripted to repeat tasks automatically.
Remote Access
Servers often only allow CLI access — no graphical interface.
02
Key Vocabulary
Words you need to know before typing your first command
Terminal
The window where you type commands.
Shell
The program that interprets your commands (e.g., Bash, Zsh).
Prompt
The text before your cursor — e.g., student@computer:~$ — means the system is ready.
Command
An instruction you give the computer — e.g., ls, cd, pwd.
Argument
What to act on — e.g., a filename or folder path passed to a command.
Flag
How to act — e.g., -l or -a modifies a command's behavior.
Directory
A folder. Same concept — different name in the CLI world.
03
Anatomy of a Command
Breaking down exactly what you type
$ ls -l /home/student/Documents │ ├── ls → the command ├── -l → a flag (detailed view) └── /home/student/Documents → the argument (where to look)
Not all commands need flags or arguments.

Running pwd on its own prints your current location. Simple commands work without extras.

04
Core Commands
The seven essential command groups every CLI user needs
pwd
Where am I?
$ pwd
/home/student
Prints your current location in the file system. Think of it as checking your GPS. If you feel lost, run pwd.
ls
What's here?
$ ls
$ ls -l
$ ls -la
Lists files and folders. -l adds details. -a includes hidden files.
cd
Move around
$ cd Documents
$ cd ..
$ cd ~
.. = parent folder  ·  ~ = home directory
mkdir
Make a folder
$ mkdir my_project
Creates a new directory. Use underscores or hyphens — avoid spaces in names.
touch · cat · echo
Create & read files
$ touch notes.txt
$ echo "Hello CLI" > notes.txt
$ cat notes.txt
> overwrites  ·  >> appends
cp · mv · rm
Copy, move & delete
$ cp notes.txt backup.txt
$ mv backup.txt Documents/
$ rm log.txt
$ rm -r my_project/
mv also renames files. rm -r deletes folders recursively.
clear · history · man
Stay organized
$ clear
$ history
$ man ls
↑ arrow = previous commands  ·  Tab = auto-complete
No recycle bin in the CLI.

rm permanently deletes files. Always ls first to confirm what you are deleting.

05
The File System Is a Tree
How Linux organizes everything on disk
/ ├── home/ │ └── student/ │ ├── Documents/ │ ├── Downloads/ │ └── notes.txt ├── etc/ └── var/ / = root (the top of the whole tree) ~ = your home folder (/home/student) cd moves you through the branches
  • Every file lives somewhere in this tree, starting at / (root)
  • ~ is shorthand for your own home directory
  • Paths starting with / are absolute — from the root
  • Paths without / are relative — from where you are now

Common Mistakes to Avoid

Spaces in filenames → use my_file.txt instead of my file.txt
Not knowing your location → run pwd before acting
Typos with rm → always ls first
Ignoring error messages → read them, they tell you exactly what's wrong
